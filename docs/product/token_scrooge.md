# Token Scrooge

## 한 줄 요약
적은 AI 토큰으로 개발 퀘스트를 완수한 사람이 이기는 웹 기반 팀 경쟁 게임.
스크루지 영감처럼 토큰을 아껴야 이긴다.

---

## 핵심 컨셉

AI에게 질문을 많이 할수록 토큰이 쌓인다. 토큰이 적을수록 순위가 높다.
퀘스트를 "잘" 풀면서도 AI 의존도를 낮추는 것이 전략의 핵심.

게임이 끝나면 내가 어디서 토큰을 낭비했는지 스스로 발견할 수 있는 리포트를 제공한다.

> "이 게임을 플레이하는 것 자체가 토큰을 소모합니다. 현명하게 쓰세요."

---

## 게임 플로우

1. 참가자가 닉네임 입력 후 입장
2. 게임 내 AI 채팅 인터페이스로만 AI 사용 가능 (외부 ChatGPT 사용 불가)
3. 퀘스트 지문을 읽고, 필요하면 AI에게 질문하면서 답안 작성
4. 퀘스트 완료 시 사용한 토큰 수 + 정답 여부로 점수 산정
5. 전 퀘스트 완료 후 리더보드 등록 + 개인 결과 리포트 확인

---

## 퀘스트 구성

코딩 문제와 설계/분석 문제를 혼합으로 구성한다.

### 코딩 퀘스트 (자동 채점)
- 함수 구현 — 주어진 스펙을 만족하는 코드 작성
- 버그 수정 — 의도적으로 심어둔 버그를 찾아서 고침
- 리팩터링 — 동작은 유지하면서 코드 개선

코드 실행은 서버 샌드박스에서 테스트케이스 통과 여부로 채점.

### 설계/분석 퀘스트 (키워드 채점)
- 트레이드오프 분석 — 두 가지 선택지의 장단점 설명
- 설계 설명 — 특정 기능을 어떻게 구현할지 서술

답안에 핵심 키워드 포함 여부 + 운영자 수동 검토로 채점.

### 트러블슈팅 퀘스트 (키워드 채점)
- 스택트레이스 분석 — 주어진 에러 로그에서 근본 원인 특정
- 슬로우 쿼리 분석 — 쿼리 실행 계획/로그를 보고 병목 지점 설명
- 장애 시나리오 재구성 — 502 에러, OOM 등 실제 장애 로그에서 원인과 흐름 재구성

이 유형이 토큰 구두쇠 게임과 가장 잘 맞는다. 로그를 스스로 읽고 원인을 좁힌 뒤 핀포인트 질문을 하면 토큰을 적게 쓰지만, AI에게 로그째로 던지고 "이거 왜 그래요?" 물어보면 토큰이 폭발한다. 실력 차이가 토큰 사용량으로 가장 극명하게 드러나는 유형이다.

---

## 스코어 계산

```
퀘스트 점수 = 정답 여부 × (기본점수 - 사용 토큰 페널티)
```

- 기본점수: 1000점
- 토큰 페널티: 100 토큰당 -10점
- 오답 시: 0점 (토큰 낭비)
- 총점 = 전 퀘스트 합산, 동점이면 토큰 적은 순으로 처리

---

## 토큰 측정 방식

참가자가 직접 외부 AI를 쓰는 게 아니라, 게임이 제공하는 채팅 인터페이스를 통해서만 AI를 사용할 수 있다.
서버가 AI API 호출을 중계하면서 입력/출력 토큰을 합산해 DB에 저장.
이 구조 덕분에 토큰 측정이 정확하고 부정 여지가 없다.

## 게임 결과 리포트

전 퀘스트 완료 후 개인 리포트 페이지를 제공한다.
정답을 알려주는 게 아니라 **패턴을 보여줘서 스스로 깨닫게** 하는 것이 목적이다.

리포트에 포함되는 내용:

- 퀘스트별 사용 토큰 수 — 어느 문제에서 가장 많이 썼는지
- 질문 횟수 vs 1위 비교 — "나: 12회 / 1위: 4회"
- 질문당 평균 토큰 — "나: 200토큰 / 1위: 80토큰"
- 질문 타임라인 — 전체 대화 흐름을 시각화하고 토큰 많이 쓴 구간 하이라이트
- 낭비 패턴 힌트 — 규칙 기반으로 자동 감지해서 표시
  - "같은 퀘스트에서 비슷한 질문을 3회 이상 반복했어요"
  - "매 질문마다 컨텍스트를 처음부터 다시 설명했어요"
  - "한 번에 물어볼 수 있는 내용을 여러 번에 나눠서 질문했어요"

리포트는 다른 사람 것도 볼 수 있게 공개 옵션을 둔다 (닉네임 기준).
1위의 질문 패턴을 보면서 "아, 이렇게 물어보면 되는구나"를 스스로 발견하게 된다.

## 팀전 모드

개인전 외에 팀전 모드를 지원한다.

### 구조
- 팀을 구성하고 같은 퀘스트 세트를 함께 풀어나감
- 팀원 전체가 하나의 AI 채팅 스레드를 공유 — 누가 무엇을 물어봤는지 실시간으로 다 보임
- 팀 토큰 = 팀원 전체 사용 토큰 합산
- 팀 순위는 팀 총점 기준, 동점이면 팀 토큰 적은 순

### 핵심 긴장감
팀원들이 병렬로 AI에게 질문할 수 있다. 빠르게 풀 수 있지만 토큰이 그만큼 빠르게 쌓인다.
"누가 먼저 스스로 생각해보고 질문할지"를 팀이 실시간으로 협의하게 된다.
채팅창에 팀원별 토큰 사용량이 보이기 때문에 "야 너 너무 많이 물어봤잖아" 같은 압박이 자연스럽게 생긴다.

### 팀 결과 리포트
개인 리포트와 별개로 팀 단위 리포트를 제공한다.
- 팀원별 기여 토큰 분포 — 누가 얼마나 사용했는지
- 병렬 질문이 발생한 구간 하이라이트 — "이 시점에 3명이 동시에 질문했어요"
- 1위 팀과의 질문 패턴 비교

## 리더보드

- 전 퀘스트 완료 시 총점 서버에 저장
- 실시간 순위표 (`/leaderboard`) 별도 제공
- 총점 높은 순 → 동점이면 토큰 적은 순으로 정렬
- 1~3위 상품 지급

## 기술 스택

- 프론트: React + 자체 채팅 UI
- AI 중계: Node.js 서버 (Anthropic SDK or OpenAI SDK)
- 채점 샌드박스: 코딩 퀘스트용 코드 실행 환경 (Judge0 or 자체 Docker)
- DB + API: Supabase
- 배포: Vercel (프론트) + Railway or Fly.io (백엔드)
