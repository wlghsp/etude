**검증용 계정 준비**

- 기존 시드는 admin 1 + member 1(`test@okestro.com`)뿐이라 "할당 O vs 할당 X" 유저를 동시 비교할 수 없는 문제 → `test2@okestro.com` member 계정 추가
- 비밀번호를 처음엔 `password123`으로 잘못 판단해 해시 생성 — 사용자가 "cloud1234!" 아니냐고 지적, bcrypt.compareSync로 실제 해시 검증해 기존 계정 비밀번호가 `cloud1234!`였음을 확인 후 test2도 동일하게 통일. **교훈: 계정 비밀번호를 가정하지 말고 사용자에게 먼저 확인하거나 실제 해시로 검증할 것.**

**curl 검증 중 발견한 버그 3건**

1. `quest.routes.ts`의 `GET /quest-sets`에 `authMiddleware` 누락 — `req.user` undefined로 500. 사용자가 가이드를 보고 직접 구현하다 빠뜨림, AI가 원인만 설명 → 사용자가 "수정 허가" 발언 후 AI가 직접 수정
2. `services/quest.ts`의 `getQuestSetsForAdmin`에서 SQL 오타 — `ON u.id = qsa_user_id`(언더스코어, 존재하지 않는 컬럼) → `qsa.user_id`로 수정. SELECT 별칭(`u.id AS userId`)과 매핑 코드(`a.user_id`)의 불일치도 함께 수정
3. `tsx src/index.ts`가 watch 모드가 아니라 파일 수정이 반영 안 되는 문제 반복 발생 — 매 수정 후 프로세스를 kill하고 `npm run dev`로 재시작해야 했음

**브라우저 검증 중 발견한 버그 2건**

4. `Admin.tsx`가 `SideNav`에 `userRole`/`onAdmin`을 전달하지 않아, 관리자 화면 진입 후 사이드바에서 "관리자" 메뉴가 사라짐 — `App.tsx`의 `<Admin>` 호출부도 `userRole` 누락되어 있었음, 둘 다 수정
5. 공개/비공개 토글·유저 할당 버튼이 반응 없음 — 콘솔 확인 결과 `Access-Control-Allow-Methods in preflight response`로 PATCH 차단. `backend/src/index.ts`의 `fastify.register(cors, { origin: true })`에 `methods` 옵션이 없었던 게 원인 → `methods: ['GET','POST','PATCH','DELETE','OPTIONS']` 명시로 해결

**curl 시나리오 → 브라우저 체크리스트로 교체**

- 가이드 초안엔 curl 기반 백엔드 검증 시나리오를 상세히 작성했으나, 사용자가 화면으로 직접 테스트하길 원해 "curl은 AI가 하고, 화면 테스트는 사용자가 한다"로 역할 분담
- 가이드의 "검증" 섹션을 curl 스크립트 대신 13단계 브라우저 체크리스트로 재작성

**UX 개선 — 접근 유저 체크박스 아코디언화**

- 세트가 많아지면(14개) 접근 유저 섹션이 테이블 맨 아래에 나타나 스크롤을 한참 내려야 하는 문제 지적받음
- 대안 3가지(아코디언/테이블 위 이동/사이드패널) 중 아코디언 방식 추천·채택 — 해당 세트 행 클릭 시 바로 아래(`<tr colSpan>`)에 펼쳐지는 구조로 변경, 재클릭 시 접힘
- 유저 수가 늘어날 걸 대비해 체크박스 목록에 `max-h-64 overflow-y-auto` 추가
- 가이드와 실제 `Admin.tsx` 모두에 반영 (Fragment key 이슈 있어 `<>` 대신 `<Fragment key={s.id}>` 사용)

**접근 유저 목록에서 admin 제외**

- 체크박스 목록에 admin 계정도 섞여 나오는 게 이상하다는 지적 → admin은 role 우회로 이미 모든 세트에 접근 가능해 개별 할당 대상이 아님이 맞다고 판단
- `services/user.ts`의 `getAllUsers`에 `WHERE role = 'member'` 조건 추가, 가이드에도 반영

**진행 방식 결정**: Phase 7h 구현/검증 완료. 세션 종료 시점에 사용자가 서버 프로세스를 직접 컨트롤하겠다고 해 AI가 백엔드를 종료함.
