### Phase 7g 라우트 분리 (신규 가이드 + 구현)

**배경**: `backend/src/index.ts`가 155줄, 라우트 9개가 순서 없이 나열되어 있어 라우팅 레이어만 도메인별로 분리.

**가이드 작성**: `guide_phase7h_route_split.md`로 초안 작성 → 이후 네이밍 재정리 과정에서 `guide_phase7g_route_split.md`로 확정 (아래 "가이드 재정리" 참고).

**구조 결정 과정**

- 처음 제안한 6개 라우트 파일(`auth`/`user`/`admin`/`quest`/`session`/`ws`) 분리안을 사용자가 그대로 채택 — "찾기 편하니까"가 이유. 파일당 코드량이 적어도 이름으로 바로 찾을 수 있는 이점을 우선.
- 라우트 파일명이 서비스 파일명과 겹치는 문제(`auth.ts` vs `routes/auth.ts`) 지적받아 `*.routes.ts` 접미사로 전체 통일 (`terminal.ts`도 `ws.ts` 대신 `terminal.routes.ts`로).
- 서비스 파일(`auth.ts`, `quest.ts`, `user.ts`, `sandbox.ts`, `terminal.ts`)도 `services/`로 이동 — "파일 이동일 뿐이니 간단하다"는 사용자 판단으로 같은 라운드에 포함.

**구현 완료 후 검증**

- `git mv`로 서비스 파일 이동 확인 (rename으로 인식됨)
- `tsc --noEmit` 통과
- `admin.routes.ts`에 남아있던 기존 버그(`PATCH /admin/users/:id/password`가 `request.params`에서 `password`를 읽어 항상 undefined) 발견 → 수정
- `services/user.ts`와 `routes/user.routes.ts`의 내용 불일치 발견 (진행/리더보드만 있고 계정 관리는 admin으로 분리되어 있음) — 이름과 실제 내용이 어긋나는 문제로 남아있음, 추후 재고 필요

**최종 구조**

```
backend/src/
  db.ts, types.ts, index.ts
  services/  auth.ts, quest.ts, user.ts, sandbox.ts, terminal.ts
  plugins/   docker.ts, auth-guard.ts
  routes/    auth.routes.ts, user.routes.ts, admin.routes.ts,
             quest.routes.ts, session.routes.ts, terminal.routes.ts
```
