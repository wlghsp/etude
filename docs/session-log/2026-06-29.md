### 도커 컨테이너 수동 정리

etude 서버 종료 후 정리되지 않은 샌드박스 컨테이너 29개 수동 제거.
`etude-db`, `k3d-etude-*` 3개만 유지.

**왜**: 컨테이너 정리 로직이 미구현 상태였기 때문에 서버를 내려도 컨테이너가 그대로 남아 있었음.

---

### 컨테이너 정리 구현 (phase5c + phase5e)

phase5c(서버 시작 시 고아 정리)와 phase5e(서버 종료 시 정리)가 가이드는 있었으나 코드에 미반영 상태였음을 확인 후 구현.

**구현 내용**

- `terminal.ts` — `createContainer` 3곳에 `Labels: { etude: 'sandbox' }` 추가 (linux, docker, k8s)
- `index.ts` — 서버 시작 시 `cleanupOrphanContainers()` 호출 (label 기반 필터링)
- `index.ts` — `onClose` 훅 + SIGTERM/SIGINT 핸들러 추가 (서버 종료 시 정리)
- `App.tsx` — `handleSetSelect`에 `setContainerId('')` 추가 (세트 진입 시 이전 containerId 초기화)

**왜 `App.tsx` 수정이 필요했나**: 서버 재시작으로 컨테이너가 사라진 뒤 같은 세트에 다시 진입하면 프론트가 이전 `containerId`를 들고 WebSocket 연결을 시도해 404 에러가 발생했음. 세트 선택 시점에 초기화하는 것이 가장 자연스러운 해결.

**검증**: 퀘스트 진입 → `docker ps --filter label=etude=sandbox` 확인 → 서버 `Ctrl+C` → 컨테이너 없음 확인.
