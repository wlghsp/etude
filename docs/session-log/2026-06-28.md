### OCI 서버 배포 계획 수립

배포 방식 결정:
- **인프라**: Terraform (OCI Always Free ARM VM 4 OCPU / 24GB RAM)
- **앱 구성**: docker-compose.prod.yml (nginx + backend + MariaDB)
- **배포**: 수동 (`git pull` + `docker compose up`) — GitHub Actions 자동 배포는 Phase 8 이후 검토
- **k8s 격리**: 현재 namespace 격리 유지. 서버 자원(24GB)으로 vcluster per user 가능하나 동시 사용자가 소수라 불필요.

**왜 Terraform**: 인프라를 코드로 관리하면 VM을 날려도 `terraform apply` 한 번으로 동일 환경 복원 가능. 설정이 문서화되는 부수 효과도 있음.

**왜 GitHub Actions 미도입**: 아직 개발 중인 상태에서 push마다 배포되면 반쯤 완성된 기능이 서버에 올라감. main push 배포는 "완성된 기능을 올린다"는 의미가 생긴 뒤에 붙이는 게 맞음.

**왜 vcluster 미도입**: OCI 서버 24GB로 vcluster per user는 기술적으로 가능하나, 동시 접속자가 소수인 사내 트레이닝 도구에 오버엔지니어링. namespace 격리로 기초 실습 수준은 충분히 커버됨.

작성 문서:
- `docs/specs/spec_phase8_deploy.md`
- `docs/guides/guide_phase8_deploy.md`

가이드 검증 결과 발견된 수정 필요 항목 (배포 시점에 반영):
- `backend/tsconfig.json` — `outDir` 주석 해제 필요 (`npm run build` 시 `dist/` 미생성 문제)
- `frontend/src/api.ts` — `:3001` 포트 제거 필요 (서버에서는 nginx가 80포트로 받음)
- `frontend/src/components/Terminal.tsx` — WebSocket `:3001` 포트 제거 필요
- `deploy/docker-compose.prod.yml` — `~` → `${HOME}` (compose 파일 내 `~` 미지원)

### Phase 순서 확정

배포 시점을 두 번 재검토 후 최종 확정:

```
Phase 7 — 사용자 인증 + 진행 추적  (다음 작업)
Phase 8 — 서버 배포 (OCI)          (인증 완료 후)
Phase 9 — 퀘스트 콘텐츠 확장 2
```

**왜 배포를 뒤로 미뤘나**: 처음엔 "서버 확보 → 인증"으로 가려다가, 인증과 퀘스트가 갖춰진 뒤에 배포해야 실사용 피드백 수집이 의미 있다는 판단으로 재결정. 배포는 급하지 않고, 팀원이 쓸 만한 수준이 된 뒤에 공개하는 게 맞음.

**배포 가이드는 미리 작성 완료** — Phase 7이 끝나면 가이드 그대로 진행 가능.

반영 문서: `etude_dev_plan.md`, `etude_dev_guide.md`, `etude_prd.md`

### CKA 퀘스트 백로그 추가

`docs/quest-backlog.md`에 CKA 준비 세트 39개 추가.
CNCF CKA 시험 도메인 (2025 최신) 기준으로 5개 영역 구성.

**왜 지금 추가했나**: 팀원들이 CKA 자격증을 준비할 수 있는 플랫폼을 만드는 것이 목표로 확정됨. k8s 퀘스트를 점진적으로 보강해가야 하므로 백로그에 미리 정리.

**구현 전제 조건**: RBAC, NetworkPolicy, CRD 등 클러스터 레벨 리소스가 포함되므로 현재 namespace 격리로는 부족. vcluster per user 도입 필요 — OCI 서버(24GB)로 이미 조건 충족.

`docs/research/k8s_cluster_isolation.md` 결론 표 업데이트:
- vcluster 전환 조건 "16GB 이상" → OCI 24GB로 이미 충족 표시 추가
