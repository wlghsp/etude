# Stitch 프롬프트 — 팀 리더보드 화면

기존 Phase 7 UI(로그인, 세트 선택, 개인 진행 현황, 실습 화면)와 동일한 디자인 시스템 위에 추가되는 화면.

---

## 제품 컨텍스트

**Etude** — 오케스트로 사내 현장 실습 트레이닝 플랫폼.
관리자가 전체 팀원의 퀘스트 세트별 진행 현황을 한눈에 파악하는 화면.

**디자인 기조**
- 다크 테마 (#111 배경, #f0f0f0 텍스트)
- 개발자 도구 느낌 (VSCode, Linear 레퍼런스)
- 폰트: monospace 계열 (JetBrains Mono 또는 동등)
- 강조색: #22c55e (완료), #3b82f6 (액션 버튼), #f59e0b (진행 중), #6b7280 (미시작)

---

## 화면: 팀 리더보드

**URL**: `/leaderboard`  
**접근 권한**: 로그인한 모든 팀원

**목적**  
전체 팀원 × 퀘스트 세트 조합의 완료 현황을 공개 리더보드로 표시.  
서로의 진행 상황을 보며 동기부여. 향후 필요 시 admin 전용으로 제한 가능.

---

### 포함 요소

**상단 헤더**
- 제목: `Team Progress` (h1 또는 h2 수준)
- 우상단: `← 돌아가기` 버튼

**요약 카드 (3~4개, 상단 행)**
- 전체 팀원 수 (예: `5명`)
- 전체 퀘스트 세트 수 (예: `14개 세트`)
- 팀 전체 평균 완료율 (예: `38%`)
- 완료 현황 좋은 팀원 (옵션: 이번 달 가장 많이 푼 사람)

**메인 테이블**

| 팀원 | 세트명 | 카테고리 | 진행률 | 상태 |
|------|--------|----------|--------|------|
| 홍길동 | 리눅스 기초 1 | 리눅스 | 10/10 (100%) | COMPLETED |
| 홍길동 | Docker 기초 | 도커 | 5/8 (63%) | IN PROGRESS |
| 김철수 | 리눅스 기초 1 | 리눅스 | 0/10 (0%) | NOT STARTED |

- **팀원 이름**: 같은 팀원의 행은 시각적으로 묶어서 표시 (rowspan 또는 배경색 교차)
- **카테고리 배지**: 리눅스🐧 / 도커🐳 / k8s☸️ 색상 구분
- **진행률 바**: 테이블 셀 안에 얇은 인라인 프로그레스 바 (4px 높이)
- **상태 배지**:
  - `COMPLETED` — #22c55e 초록
  - `IN PROGRESS` — #3b82f6 파랑 또는 #f59e0b 주황
  - `NOT STARTED` — #6b7280 회색

**팀원별 그룹핑 힌트**
- 같은 팀원의 여러 세트 행이 연속으로 나열됨
- 팀원 이름 셀은 첫 행에만 표시하거나, 행 사이 구분선으로 그룹 구분

---

### 레이아웃

- 전체 너비 레이아웃 (max-width 1000px, 중앙 정렬)
- 개인 진행 현황(`/progress`)과 동일한 헤더/네비게이션 구조
- 테이블은 스크롤 가능 (팀원 수 × 세트 수가 많아질 수 있음)

---

### 화면 간 이동 흐름 (참고)

```
/ (세트 선택)
  ├─ Progress Status 클릭 → /progress (내 진행 현황)
  └─ Leaderboard 클릭 → /leaderboard  ← 이 화면 (모든 팀원 접근)
```
