# Stitch 프롬프트 — Phase 7 UI 설계 요청

Phase 7(사용자 인증 + 진행 추적) 구현 전 화면 설계를 위한 Stitch 요청 문서.

---

## 제품 컨텍스트

**Etude** — 오케스트로 사내 현장 실습 트레이닝 플랫폼.
팀원이 브라우저에서 리눅스/Docker/k8s 퀘스트를 풀고 채점받는 도구.

**디자인 기조**
- 다크 테마 (#111 배경, #f0f0f0 텍스트)
- 군더더기 없는 개발자 도구 느낌 (VSCode, Linear 레퍼런스)
- 폰트: monospace 계열
- 현재 UI 예시:
  - 세트 선택: 카테고리 아코디언 (리눅스🐧 / 도커🐳 / k8s☸️)
  - 실습 화면: 좌측 40% 퀘스트 패널 + 우측 60% 터미널 (xterm.js, 검정 배경 초록 텍스트)

---

## 요청 화면 3개

---

### 화면 1. 로그인 페이지

**URL**: `/login`

**목적**: 사내 구성원 인증. 이메일 + 비밀번호 입력.

**포함 요소**
- 상단: `OKESTRO TRAINING` 레이블 (소문자, 자간 넓게) + `Etude` 타이틀 (32px bold)
- 이메일 입력 필드 (placeholder: `이름@okestro.com`)
- 비밀번호 입력 필드
- 로그인 버튼
- 에러 메시지 표시 영역 (잘못된 이메일/비밀번호)

**레이아웃 힌트**
- 화면 중앙 카드 (max-width 400px)
- 현재 세트 선택 화면과 같은 중앙 정렬 구조로 통일감 유지

---

### 화면 2. 세트 선택 화면 (기존 + 진행률 추가)

**URL**: `/` (로그인 후 진입)

**현재 구조** (변경 없는 부분)
- 카테고리 아코디언 (리눅스 / 도커 / k8s)
- 각 카테고리 클릭 시 세트 목록 펼쳐짐

**추가할 요소**
- 우상단: 로그인한 사용자 이름 + 로그아웃 버튼
- 각 세트 항목에 진행률 표시:
  - `n/m 완료` 텍스트 (예: `3/12 완료`)
  - 또는 얇은 프로그레스 바 (세트 카드 하단)
  - 모두 완료된 세트는 체크 배지 또는 색상 구분

**레이아웃 힌트**
- 세트 카드 우측에 진행률 정보 배치
- 완료 세트는 카테고리 아이콘 옆에 작은 ✓ 표시

---

### 화면 3. 진행 현황 페이지 (대시보드)

**URL**: `/progress`

**목적**: 내가 어떤 세트를 얼마나 풀었는지 한눈에 파악.

**포함 요소**
- 상단 요약 카드 (3개):
  - 전체 퀘스트 수
  - 완료한 퀘스트 수
  - 완료율 (%)
- 세트별 진행 현황 테이블 또는 카드 목록:
  - 세트 이름
  - 카테고리 (리눅스 / 도커 / k8s)
  - 진행률 바 + `n/m`
  - 완료 여부 (완료 / 진행 중 / 미시작)
- 세트 클릭 시 해당 세트로 바로 진입

**레이아웃 힌트**
- 세트 선택 화면에서 링크로 진입 (우상단 `진행 현황` 버튼)
- 전체 너비 레이아웃 (max-width 800px) 또는 세트 선택과 동일한 520px 중앙 카드

## 화면 간 이동 흐름

```
/login
  └─ 로그인 성공
       └─ / (세트 선택)
            ├─ 세트 클릭 → /quest (실습 화면, 기존 유지)
            └─ 진행 현황 버튼 → /progress
```

### 화면 4. 실습 화면 — 퀘스트 패널 개선

**URL**: `/quest` (세트 선택 후 진입)

**현재 구조**
- 좌측 40%: 퀘스트 패널 (제목, 설명, 힌트 보기, 풀이 보기, 채점하기 버튼, 이전/다음/홈으로)
- 우측 60%: xterm.js 터미널 (검정 배경, 초록 텍스트)

**현재 채점 결과 표시**
- 성공: 초록 배경 박스 "✅ 성공!"
- 실패: 빨간 배경 박스 "❌ 아직이에요. 다시 시도해보세요."

**추가/변경할 요소**

1. **환경 리셋 버튼**
   - 위치: 퀘스트 패널 하단 버튼 영역 (홈으로 버튼 옆)
   - 동작: 현재 컨테이너를 삭제하고 새 컨테이너로 교체 (터미널 재연결)
   - 용도: 실습 중 환경이 꼬였을 때 탈출구. k8s/docker 세트에서 특히 중요.
   - UI: "환경 리셋" 또는 "↺ 리셋" 레이블, 실수 방지를 위해 확인 클릭 또는 홀드 방식

2. **채점 성공 시 자동 다음 퀘스트 유도**
   - 현재: 성공 메시지만 표시, 사용자가 직접 "다음" 버튼 클릭
   - 개선: 성공 시 "다음 퀘스트로 →" 버튼을 채점 결과 옆에 강조해서 표시
   - 마지막 퀘스트 성공 시: "세트 완료! 🎉" 메시지 + 홈으로 버튼 강조

3. **퀘스트 진행 표시 개선**
   - 현재: "Quest 3 / 12" 텍스트만
   - 개선: 상단에 얇은 프로그레스 바 (완료된 퀘스트 수 시각화)
   - 완료된 퀘스트는 바 위에 체크 마크 또는 색상으로 구분

**레이아웃 힌트**
- 퀘스트 패널은 스크롤 가능 (긴 설명 대응)
- 버튼 영역은 패널 하단에 고정 (스크롤해도 항상 보임)
- 터미널 영역은 변경 없음
