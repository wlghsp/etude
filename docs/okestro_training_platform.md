# Etude — 오케스트로 현장 실습 트레이닝 플랫폼

## 네이밍

**Etude** (에튀드)

쇼팽 에튀드처럼, 특정 기술을 반복 훈련하기 위해 설계된 연습곡을 의미한다.
악보(문서)를 읽는 것만으로는 연주를 못 하듯, 인수인계 문서를 읽는 것만으로는 현장 대응을 못 한다.
직접 손으로 반복해야 는다 — 이것이 Etude의 철학이다.

어원적으로 프랑스어 "공부, 연구, 습작(Study)"을 뜻한다. 음악적 의미(반복 연습곡)와 어원적 의미(공부, 연구)가 두 겹으로 쌓여 트레이닝 플랫폼 이름으로 더할 나위 없이 맞아떨어진다.

오케스트로 제품군(콘트라베이스, 비올라 등)의 악기 네이밍 톤과 자연스럽게 이어진다.

---

## 한 줄 요약
실무자가 현장 작업과 문제를 입력하면 AI가 트레이닝 퀘스트로 변환해주는 사내 실습 교육 플랫폼.
특정 인원에게 쏠린 현장 지식을 조직 전체로 분산시키는 것이 목적이다.

---

## 왜 만드는가

공공기관 납품 및 현장 작업이 필요한 구조에서, 실제로 무엇을 어떻게 하는지 아는 인원이 소수에 집중되어 있다.
다른 구성원들은 현장에 나가도 무슨 작업인지, 어떻게 대응하는지 모른다.

KodeKloud 같은 범용 실습 플랫폼은 있지만 우리 회사 현장 상황을 그대로 반영한 커스텀 퀘스트를 만들 수 없다.
Etude는 실무자가 직접 현장 지식을 입력하면 다른 구성원이 그것을 실습으로 학습하는 구조를 만든다.

## 핵심 기능

### 1. 퀘스트 생성기
실무자가 실제 작업이나 장애 사례를 텍스트로 입력하면 AI가 트레이닝 퀘스트로 자동 변환한다.
- 입력: "OO 기관 서버에서 Nginx 설정 변경 후 502 발생했고 이렇게 해결했다"
- 출력: 동일한 상황을 재현한 실습 퀘스트 + 채점 기준

인수인계 문서도 입력 소스가 될 수 있다. 문서를 넣으면 AI가 핵심 작업을 퀘스트로 변환하고, 인수자는 읽는 것에 그치지 않고 직접 수행하며 숙지한다.

실무자가 퀘스트 생성에 기여할수록 플랫폼의 퀘스트 풀이 현장 밀착형으로 쌓인다.

### 퀘스트 품질 검증 파이프라인
AI가 생성한 퀘스트는 바로 등록되지 않고 검증 단계를 거친다.

실무자 입력 → AI 퀘스트 생성 → AI 에이전트 1차 리뷰 → 실무자 최종 승인 → 퀘스트 풀 등록

AI 에이전트 1차 리뷰에서 확인하는 것:
- 채점 기준이 명확하고 일관성이 있는지
- 지문이 모호하거나 오해의 여지가 없는지
- 난이도가 의도한 수준에 맞는지
- 재현 환경 구성이 실제로 가능한지

1차 리뷰를 통과한 퀘스트만 실무자에게 전달되므로, 실무자는 퀘스트를 처음부터 다시 풀 필요 없이 최종 확인만 하면 된다. 실무자의 기여 부담을 줄이면서 품질을 확보하는 구조다.

### 2. 브라우저 터미널 실습 환경
KodeKloud처럼 브라우저에서 바로 터미널을 열고 실습할 수 있다.
Docker 기반 샌드박스로 각 퀘스트마다 독립된 환경을 제공한다.
- 리눅스 명령어 기초
- Docker / Kubernetes 작업
- Nginx, DB 설정 변경
- 실제 장애 상황 재현 및 트러블슈팅

## 퀘스트 유형

### 기초 실습
- 리눅스 명령어 — 파일 조작, 권한 설정, 프로세스 관리
- 네트워크 확인 — ping, curl, netstat, ss로 상태 점검
- 서비스 관리 — systemctl, journalctl로 서비스 제어 및 로그 확인

### 트러블슈팅
- 스택트레이스/에러 로그에서 근본 원인 특정
- 슬로우 쿼리, 높은 CPU/메모리 사용률 원인 분석
- 실제 장애 시나리오 재현 (502, OOM, CrashLoopBackOff 등)

### 현장 작업 시뮬레이션
- 실무자가 제출한 실제 작업을 퀘스트화한 것
- 기관별 환경 차이를 반영한 커스텀 시나리오
- 정해진 절차대로 수행하는 것과 응용 대응 두 가지 유형

### 인수인계 퀘스트
- 인수인계 문서 기반으로 AI가 자동 생성
- 핵심 작업을 직접 수행해보는 형태로 변환

## 향후 확장 가능한 문제 유형

MVP는 터미널 퀘스트에 집중한다. 이후 플랫폼이 안정화되면 아래 유형으로 확장을 고려할 수 있다.

- 객관식/OX — 구현 난이도 낮음. Badge 준비용으로 활용 가능
- 코드 에디터 + 실행 — Judge0 같은 채점 엔진 필요. 개발 직군 퀘스트에 유용
- 서술형 — AI 채점 필요. 설계/분석 퀘스트에 적합
- 다이어그램/설계 제출 — 채점 기준 정의가 어려움. 수동 검토 병행 필요

각 유형마다 채점 방식과 구현 복잡도가 다르므로 단계적으로 추가한다.

## 트레이닝 세트 구조

퀘스트는 개별 문제 단위이고, 트레이닝 세트는 퀘스트를 목적에 맞게 묶은 단위다.

### 역할
- **트레이니** — 세트를 선택해서 자율 학습하거나, 관리자가 할당한 세트를 기한 내에 완료
- **관리자/팀장** — 세트를 구성원에게 할당하고 기한을 설정, 완료 현황을 모니터링

### 세트 분류 기준
세트 하나에 여러 태그가 붙는 구조로 유연하게 관리한다.

직군별 태그 — 인프라, 개발, 운영
난이도별 태그 — 입문, 중급, 고급
상황별 태그 — 온보딩용, 인수인계용, badge 1단계/2단계/3단계용, 채용 전형용

### 세트 구성 예시
- 인프라 신입 온보딩 세트 — 리눅스 기초 + 네트워크 확인 + 서비스 관리 (입문, 온보딩용)
- 장애 대응 심화 세트 — 트러블슈팅 + 현장 작업 시뮬레이션 (고급, 인프라)
- badge 2단계 준비 세트 — 트러블슈팅 퀘스트 모음 (중급, badge 2단계용)

## 활용 시나리오

Etude는 구성원의 전 생애주기에 걸쳐 사용된다.

### 채용 시 — 실무 밀착형 실기 전형
코딩테스트처럼 플랫폼에서 지원자의 실력을 검증하는 전형으로 활용한다.
일반적인 알고리즘 문제가 아니라 우리 회사 현장과 직결된 퀘스트로 검증하기 때문에 채용 후 미스매치가 줄어든다.

- 직군별 채용 퀘스트 세트 — 인프라 엔지니어, 개발자, 운영 담당 등 포지션에 맞는 퀘스트 제공
- 정답 여부만이 아니라 접근 방식 평가 — AI를 어떻게 활용했는지, 어떤 순서로 문제에 접근했는지 리포트로 확인
- 면접 전 사전 검증 — 서류 통과 후 플랫폼 퀘스트를 과제로 주고 결과를 면접 자료로 활용

### 입사 직후 — 온보딩 소프트랜딩
신규 입사자의 소프트랜딩을 지원한다.
기존 오리엔테이션이 PPT와 문서 읽기로 끝나는 것과 달리, Etude와 함께하면 입사 첫 주부터 실제 현장 작업을 손으로 체험할 수 있다.

- 직군별 온보딩 퀘스트 트랙 — 인프라, 개발, 운영 등 역할에 맞는 퀘스트 세트 제공
- 단계적 난이도 — 리눅스 기초부터 시작해서 실제 현장 작업 시뮬레이션까지
- 온보딩 완료 체크리스트 — 퀘스트 수행 결과로 자동 집계, 담당자가 진행 상황 모니터링
- 인수인계 모드와 연계 — 전임자가 남긴 인수인계 문서가 신규 입사자의 온보딩 퀘스트로 바로 연결

### 재직 중 — 현장 트레이닝 및 인수인계
실무자가 현장 작업과 장애 사례를 입력하면 다른 구성원이 그것을 실습으로 학습한다.
인수인계 문서를 플랫폼에 입력하면 AI가 관련 퀘스트를 자동 생성하고, 인수자는 직접 수행하며 숙지한다.

### 승진 시 — Digital Badge 연계
현재 사내 승진 시험인 digital badge와 연결한다.

1단계 badge — 객관식 시험 + 기초 실습 퀘스트 병행
2단계 badge — 트러블슈팅 퀘스트 + 현장 작업 시뮬레이션
3단계 badge — 논문/발표 주제를 플랫폼에서 직접 구현하고 결과 제출

객관식(현재) → 실습(Etude) → 발표 사이의 빈 중간 단계를 채운다.
badge 취득이 트레이닝 완료와 연결되므로 참여 동기가 자연스럽게 생긴다.

## AI 활용 방침 및 피드백

AI를 쓰면서 풀어도 되지만, 목표는 AI 없이 풀 수 있을 때까지 반복 트레이닝하는 것이다.

### AI 사용 피드백 리포트
퀘스트를 풀 때마다 AI 사용 패턴을 기록하고, 완료 후 피드백을 제공한다.
정답을 알려주는 게 아니라 어떻게 AI를 사용했는지 패턴을 보여줘서 스스로 깨닫게 하는 것이 목적이다.

리포트에 포함되는 내용:
- 이번 퀘스트에서 사용한 토큰 수와 질문 횟수
- 낭비 패턴 자동 감지 — "같은 내용을 3회 이상 반복 질문했어요", "로그를 그대로 붙여넣고 물어봤어요"
- 반복 수행 시 성장 추이 — 같은 퀘스트를 다시 풀 때 AI 의존도가 줄었는지 수치로 표시
- 동료 비교 — 같은 퀘스트를 푼 다른 구성원 대비 AI 사용량 위치

이 피드백을 통해 "AI에게 어떻게 물어보는 게 효율적인가"를 체득하게 된다.
결국 AI를 잘 쓰는 능력 자체가 트레이닝의 부산물로 길러진다.

## 학습 경로 및 진행 추적

- 구성원별 퀘스트 완료 현황 대시보드
- 퀘스트 유형별 숙련도 추적 — 처음엔 AI 도움, 나중엔 단독 수행
- badge 단계별 필수 퀘스트 이수 현황
- 인수인계 완료율 — 문서 확인 여부가 아니라 퀘스트 수행 여부 기준

## 기술 스택

- 프론트: Vue.js
- 터미널 실습 환경: Docker 기반 샌드박스 (xterm.js로 브라우저 터미널 제공)
- 퀘스트 생성 AI: Claude API (실무자 입력 → 퀘스트 자동 변환)
- DB: MariaDB (사내 기존 운영 스택)
- 채점: 코드/명령어 실행 결과 자동 검증 + 일부 수동 검토
- 배포: 사내 서버 (공공기관 납품 특성상 데이터 외부 유출 방지)
