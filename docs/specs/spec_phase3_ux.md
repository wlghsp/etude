# Phase 3 명세 — UX 개선

## 목표

Phase 2까지 완성된 MVP를 직접 써보면서 발견된 UX 문제 3가지를 수정한다.

---

## 개선 항목

### 3-1. 채점 중 로딩 표시

**문제:** 채점하기 버튼 클릭 후 응답이 올 때까지 아무 반응이 없어 답답함.

**해결:**
- 채점 요청 중 버튼 텍스트를 "채점 중..." 으로 변경
- 버튼 disabled 처리 (중복 클릭 방지)

**검증:** 채점하기 클릭 → 버튼이 "채점 중..."으로 바뀌고 disabled → 결과 도착 후 원래대로

---

### 3-2. 퀘스트 완료 후 다음 단계 안내

**문제:** 마지막 퀘스트를 성공해도 아무 안내 없이 끝남. 완료됐는지 알 수 없음.

**해결:**
- 마지막 퀘스트 채점 성공 시 완료 메시지 표시
- 메시지: "🎉 모든 퀘스트를 완료했습니다!"

**조건:**
- 마지막 퀘스트(`index === total - 1`) + 채점 통과 시에만 표시
- 중간 퀘스트 성공 시에는 미표시

**검증:** Quest 2 채점 성공 → 완료 메시지 표시

---

### 3-3. 터미널 스타일 개선

**문제:** 터미널 폰트/여백이 날것 상태라 가독성이 낮음.

**해결:**
- xterm.js 옵션: `fontSize: 14`, `fontFamily: 'monospace'`, `lineHeight: 1.2`
- 터미널 컨테이너 좌우 padding 추가 (`padding: '4px'`)
- 배경색 xterm 기본값(`#000`) 유지 — 별도 변경 없음

**검증:** 터미널 텍스트가 이전보다 읽기 편한지 육안 확인
